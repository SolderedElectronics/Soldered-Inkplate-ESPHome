The `inkplate_spi` display platform supports Soldered Inkplate e-paper displays that communicate over SPI.

Supported models:

| Model | Resolution | Colors | Min Refresh |
|-------|------------|--------|-------------|
| `inkplate13` | 1200×1600 | 6 (ACeP Spectra 6) | 30 s |
| `inkplate6color` | 600×448 | 6 (ACeP) | — |
| `inkplate2` | 104×212 | 2 (B/W) | — |

The component uses [SPI](/components/spi), so you need an `spi:` section in your configuration.
PSRAM is required on all supported boards.

## Minimal configuration example

```yaml
spi:
  clk_pin: GPIO18
  mosi_pin: GPIO23

display:
  - platform: inkplate_spi
    model: inkplate6color
    id: my_display
    update_interval: 60s
    lambda: |-
      it.fill(Color::WHITE);
      it.print(10, 10, id(my_font), Color::BLACK, "Hello, Inkplate!");
```

All GPIO pins are pre-configured per model and do not need to be specified unless you need to override them.

## Configuration variables

- **model** (**Required**, string): The Inkplate model. One of `inkplate13`, `inkplate6color`, `inkplate2`.
- **id** (*Optional*, [ID](/guides/configuration-types#id)): Manually specify the ID used for code generation.
- **update_interval** (*Optional*, [Time](/guides/configuration-types#time)): Interval to redraw the screen.
  Defaults to `1min` if `lambda` or `pages` is configured, otherwise unset (manual updates only).
  Inkplate 13 enforces a minimum of `30s` due to ACeP panel requirements.
- **full_update_every** (*Optional*, int): Number of updates before a full refresh is forced when partial updates are active. Defaults to `1`.
- **lambda** (*Optional*, [lambda](/automations/templates#config-lambda)): Lambda for rendering display content.
  See [Display Rendering Engine](/components/display#display-engine).
- **pages** (*Optional*, list): Show pages instead of a single lambda. See [Display Pages](/components/display#display-pages).
- **rotation** (*Optional*, int): Rotate the display. One of `0°` (default), `90°`, `180°`, `270°`.
- **spi_id** (*Optional*, [ID](/guides/configuration-types#id)): SPI bus to use. Required if multiple SPI buses are configured.

### Pin overrides

All pins below have model-specific defaults and do not need to be set in most configurations.
Override only if your hardware differs from the standard Inkplate pinout.

- **pin_rst** (*Optional*, [Pin](/guides/configuration-types#pin)): Reset pin (output).
- **pin_dc** (*Optional*, [Pin](/guides/configuration-types#pin)): Data/Command select pin (output).
- **pin_busy** (*Optional*, [Pin](/guides/configuration-types#pin)): Busy signal pin (input).
- **pin_pwr_en** (*Optional*, [Pin](/guides/configuration-types#pin)): Power enable pin (output).
- **pin_cs** (*Optional*, [Pin](/guides/configuration-types#pin)): Chip select pin (output). Used by `inkplate6color` and `inkplate2`.
- **pin_cs_m** (*Optional*, [Pin](/guides/configuration-types#pin)): Master chip select pin (output). Inkplate 13 only.
- **pin_cs_s** (*Optional*, [Pin](/guides/configuration-types#pin)): Slave chip select pin (output). Inkplate 13 only.
- **pin_bs0** (*Optional*, [Pin](/guides/configuration-types#pin)): Bus select 0 pin (output). Inkplate 13 only.
- **pin_bs1** (*Optional*, [Pin](/guides/configuration-types#pin)): Bus select 1 pin (output). Inkplate 13 only.

### Default GPIO assignments per model

| Pin key | inkplate13 | inkplate6color | inkplate2 |
|---------|------------|----------------|-----------|
| `pin_rst` | GPIO4 | GPIO19 | GPIO19 |
| `pin_dc` | GPIO14 | GPIO33 | GPIO33 |
| `pin_busy` | GPIO7 | GPIO32 | GPIO32 |
| `pin_pwr_en` | GPIO21 | — | — |
| `pin_cs` | — | GPIO27 | GPIO27 |
| `pin_cs_m` | GPIO42 | — | — |
| `pin_cs_s` | GPIO39 | — | — |
| `pin_bs0` | GPIO6 | — | — |
| `pin_bs1` | GPIO5 | — | — |

## Inkplate 2

A 2.13" black-and-white display on an ESP32 module.

> [!NOTE]
> Use `rotation: 270°` to match the board's natural orientation. Without this, content will appear rotated relative to the silkscreen and physical layout of the board.

```yaml
esphome:
  name: inkplate2
  on_boot:
    priority: -100
    then:
      - component.update: my_display

esp32:
  framework:
    type: esp-idf
  variant: esp32

psram:
  mode: quad
  speed: 40MHz

spi:
  id: spi_display
  clk_pin: GPIO18
  mosi_pin: GPIO23

font:
  - file: "gfonts://Roboto"
    id: font_body
    size: 12

display:
  - platform: inkplate_spi
    model: inkplate2
    id: my_display
    spi_id: spi_display
    rotation: 270°
    update_interval: never
    lambda: |-
      const int W = it.get_width();   // 212 after 270° rotation
      const int H = it.get_height();  // 104 after 270° rotation

      it.fill(Color::WHITE);
      it.filled_rectangle(0, 0, W, 20, Color::BLACK);
      it.print(8, 2, id(font_body), Color::WHITE, "Hello from Inkplate 2");
```

## Inkplate 6 Color

A 6" 6-color ACeP display on an ESP32 module.

> [!NOTE]
> Use `rotation: 180°` to match the board's natural orientation. Without this, content will appear rotated relative to the silkscreen and physical layout of the board.

```yaml
esphome:
  name: inkplate6color
  on_boot:
    priority: -100
    then:
      - component.update: my_display

esp32:
  framework:
    type: esp-idf
  variant: esp32

psram:
  mode: quad
  speed: 40MHz

spi:
  id: spi_display
  clk_pin: GPIO18
  mosi_pin: GPIO23

font:
  - file: "gfonts://Roboto"
    id: font_title
    size: 44

display:
  - platform: inkplate_spi
    model: inkplate6color
    id: my_display
    spi_id: spi_display
    rotation: 180°
    update_interval: 60s
    lambda: |-
      it.fill(Color::WHITE);
      it.filled_rectangle(0, 0, it.get_width(), 60, Color::BLACK);
      it.print(14, 8, id(font_title), Color::WHITE, "Inkplate 6 Color");
```

### Available colors on 6-color models

Colors are matched to the nearest supported palette entry:

```cpp
auto BLACK  = Color(0,   0,   0  );
auto WHITE  = Color(255, 255, 255);
auto RED    = Color(255, 0,   0  );
auto GREEN  = Color(0,   200, 0  );
auto BLUE   = Color(0,   0,   255);
auto YELLOW = Color(255, 255, 0  );
```

### LVGL integration

The component is compatible with [LVGL](/components/lvgl/). Use `update_interval: never` and trigger
`component.update` manually to push the rendered buffer to the panel.

```yaml
display:
  - platform: inkplate_spi
    model: inkplate6color
    id: my_display
    update_interval: never

lvgl:
  displays:
    - my_display
  widgets:
    - label:
        text: "Hello LVGL"
        align: CENTER

interval:
  - interval: 60s
    then:
      - if:
          condition:
            lambda: 'return !id(my_display).is_busy();'
          then:
            - component.update: my_display
```

## Inkplate 13

A 13.3" 6-color ACeP (Spectra 6) display on an ESP32-S3 module.
Uses two SPI chips in parallel (master + slave) for full-width rendering.

> [!NOTE]
> Use `rotation: 270°` to match the board's natural orientation. Without this, content will appear rotated relative to the silkscreen and physical layout of the board.

> [!WARNING]
> The ACeP panel requires a **minimum 30 seconds** between refreshes. The component enforces this — setting
> `update_interval` below `30s` will cause a configuration error.

```yaml
esphome:
  name: inkplate13
  on_boot:
    priority: -100
    then:
      - component.update: my_display

esp32:
  framework:
    type: esp-idf
  variant: esp32s3

psram:
  mode: octal
  speed: 80MHz

# Inkplate 13 has multiple SPI peripherals — pin the display SPI explicitly.
spi:
  id: spi_display
  clk_pin: GPIO38
  mosi_pin: GPIO40
  miso_pin: GPIO41

font:
  - file: "gfonts://Roboto"
    id: font_title
    size: 72

display:
  - platform: inkplate_spi
    model: inkplate13
    id: my_display
    spi_id: spi_display
    rotation: 270°
    update_interval: 60s
    lambda: |-
      it.fill(Color::WHITE);
      it.filled_rectangle(0, 0, it.get_width(), 86, Color::BLACK);
      it.print(28, 6, id(font_title), Color::WHITE, "Inkplate 13");
```

### Partial updates

The Inkplate 13 supports partial region refresh. Use `display_partial(x, y, width, height)` to redraw
a subsection of the screen without a full refresh. Always check `is_busy()` before triggering.

```yaml
interval:
  - interval: 30s
    then:
      - lambda: |-
          if (id(my_display).is_busy()) return;
          id(my_display).filled_rectangle(370, 1160, 1200, 40, Color::WHITE);
          id(my_display).print(378, 1164, id(font_small), Color::BLACK, "Updated");
          id(my_display).display_partial(370, 1160, 1200, 40);
```

> [!NOTE]
> Logical canvas dimensions depend on rotation. At `rotation: 270°` the canvas is 1600×1200 (width×height).

## GPIO expander

The Inkplate 6 Color and Inkplate 13 include an onboard [PCA6416A](/components/pca6416a/) GPIO expander connected via I2C.
This expander provides additional I/O pins for controlling peripherals such as the onboard LED.
The Inkplate 2 does not have a GPIO expander.

Add an `i2c:` and `pca6416a:` section to your configuration to use it:

**Inkplate 6 Color** (I2C on GPIO21/GPIO22):

```yaml
i2c:
  sda: GPIO21
  scl: GPIO22
  frequency: 400kHz

pca6416a:
  - id: pcal
    address: 0x20
```

**Inkplate 13** (I2C on GPIO8/GPIO9):

```yaml
i2c:
  sda: GPIO8
  scl: GPIO9
  frequency: 400kHz

pca6416a:
  - id: pcal
    address: 0x20
```

Once configured, expander pins are available as standard ESPHome GPIO pins. Example — toggling the onboard LED:

```yaml
switch:
  - platform: gpio
    name: "Inkplate LED"
    pin:
      pca6416a: pcal
      number: 0
      mode:
        output: true
```

## Runtime API

These methods are available on the display object in lambdas and automations:

- **`is_busy()`** — Returns `true` if a refresh is in progress. Use before triggering updates to avoid overlapping transfers.
- **`display_partial(x, y, width, height)`** — Pushes a rectangular region to the panel without a full refresh. Inkplate 13 only.
